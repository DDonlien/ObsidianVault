#### 武器
- 武器的VFX由3层构成：枪口；弹道；击中。每层单独制作，由程序组合
	- 喷射器类型的武器可能没有枪口VFX，但会由程序负责处理
##### 枪口
- 小型子弹对应普通枪械，应该兼容单发和超高频连发
- 大型子弹对应火箭弹之类的发射效果，也可能复用在弹道类技能
	- [ ] 动能枪口 - 小
	- [ ] 热能枪口 - 小
	- [ ] 电磁枪口 - 小
##### 弹道
武器的每个类别，根据属性不同，会有不同的VFX，但大致复用
- 近战武器挥动尾迹
	- [ ] 动能武器
	- [ ] 热能武器
	- [ ] 电磁武器
- 远程武器 - 激光
	- 激光武器应该包含
	- [ ] 动能激光
	- [ ] 热能激光
	- [ ] 电磁激光
- 远程武器 - 弹道
	- 弹道效果应该包含一个有尾迹的弹头，以及一道弹道烟雾
	- 小型子弹对应普通枪械；大型子弹对应火箭弹之类，也可能复用在弹道类技能
	- 弹道效果应该兼容非直线弹道，用以实现导弹抛物线效果
	- [ ] 动能子弹 - 小型
	- [ ] 热能子弹 - 小型
	- [ ] 电磁子弹 - 小型
	- [ ] 动能子弹 - 大型
	- [ ] 热能子弹 - 大型
	- [ ] 电磁子弹 - 大型
- 远程武器 - 范围
	- 典型的范围伤害效果：火焰喷射器（但这个火可能是电离蓝火之类的）
	- 范围VFX应该兼容持续释放（实际配置过程中，应该就是超高频的反复释放）
	- 范围VFX应该兼容0~360°的释放范围和不同距离
	- [ ] 动能范围（有点抽象）
	- [ ] 热能范围
	- [ ] 电磁范围 
##### 击中
武器在集中敌人后，首先自己应该造成击中的VFX

受击打对象会根据自己的状态，再显示一次VFX

例如：动能子弹击中目标后，应该先自己生成子弹击中的VFX，对象如果是有护甲角色，应该生成护甲被击打的VFX；如果是无护甲生物角色，应该生成喷血VFX；如果是无护甲机械角色，应该生成金属受击VFX等。

需要的武器击中VFX包括
- [ ] 动能命中 - 小
- [ ] 热能命中 - 小
- [ ] 电磁命中 - 小

##### 瞄准VFX
技能的VFX提醒应该有3个步骤组成：
1. UI层面的瞄准，在释放前使用，半透明，仅玩家自己可见
2. 释放后的VFX
3. Loading期间的VFX
- [ ] 指向单点技能
	- 不涉及长度
	- 不涉及蓄力？
- [ ] 非指向线性技能
	- 长度自适应
	- 宽度自适应
	- 兼容蓄力
		- 下图这个比较含蓄，其实我觉得一开始有瞄准线，然后瞄准线像刮刮卡一样变深色更明显
		- ![[1月3日.gif]]
	- 兼容无限/超长距离情况（仅提示方向、不提示距离；或者用哈迪斯的方案，展示一个全图的指引）
	- 无限举例也可以
- [ ] 非指向扇形技能
	- 长度自适应
	- 兼容蓄力
	- 宽度自适应
- [ ] 非指向选点技能
	- 半径自适应
	- 兼容蓄力
##### 提示VFX
提示VFX主要用于提示该技能即将释放，并展示范围
提示VFX与瞄准VFX应该一一对应，事实上，可以复用大部分素材，只做材质变换

##### 释放VFX
各类技能VFX需要N个不同属性的变种
- DEBUFF
	- DEBUFF应该有2种状态的VFX：
		- 持续状态，用于提示玩家该单位正受到该DEBUFF，例如灼烧的单位应该遍身有火焰（之类的）
		- 触发状态，DEBUFF在满足效果情况下触发时的DEBUFF，例如灼烧是每秒造成伤害，则造成伤害的那一下应该有单独的VFX
	- [ ] 动能DEBUFF：重伤
	- [ ] 电磁DEBUFF：麻痹
	- [ ] 热能DEBUFF：灼烧
- 弹道
	- 弹道类技能原则上应该每个有各自的特效，但DEMO阶段可以先结合子弹弹道复用（比如组合小、大两类），实现和子弹区别即可
下列两类特效在常规游戏比较常见，但考虑到我们游戏的科幻属性，复用性没有那么强，可以稍后制作
- 天体坠落
- 地面升腾
