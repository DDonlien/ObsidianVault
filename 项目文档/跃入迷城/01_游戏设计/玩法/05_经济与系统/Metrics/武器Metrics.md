
**武器Metrics**

**基础**

1. 玩家可以装备2把武器
2. 每把武器附带1个主动技能
3. 不同种类的武器，可能附带的主动技能不同，每个种类的武器从同一个主动技能池中随机获得一个技能
4. *智脑也属于一类武器*
5. 武器攻击会造成“**附带属性的基础伤害**”和“**异常积蓄**”
6. 由于游戏中的生命中随进度发展而提升，但异常耐性基本会维持不变，因此导致以下几个设计
1. 大部分伤害增幅效果，如果没有特别说明，不会增加异常积蓄
2. 属性补正的武器伤害不会增加异常积蓄
7. 上述同样的情况也适用于韧性削减

**获取、升级**

1. 武器可以升级，通过*打印机*升级持有的武器
2. 武器升级时，主动技能也会升级
3. 武器由武器打印机提供，每次刷新提供1个，刷新消耗信用点，*获取不用*
4. 不同的关卡层级，会提供不同等级的武器候选项，基本规则类似于：
- 在Lv5，会刷新Lv4、Lv5、Lv6的武器
- *具体刷新概率待定*

**数值平衡**

**体力价值**

数值定义：

- 每动伤害Dpa
- 攻击时长Ta
- 标准体力价值 =肘击伤害 / 肘击体力消耗 = 2

现在的调整设计有2个思路：

1. 每动伤害越高，体力性价比越低，因为高单动伤害的武器更容易通过有限的输出窗口造成伤害
2. 攻击速度越快，体力性价比越低，因为高攻速武器可以在有限的输出窗口内更稳定的造成伤害和冲击力压制

我们用如下公式平衡上述2个思路：

- 期望体力**Cstamina** = **Dpa** / [2 x ( **Dpa** / 20 ) ^ (-0.2)] x (**Ta**) ^ 0.3

带保护的Excel公式如下：

- IFERROR(ROUND(AR4 / (2 \* 1 \* (AR4/20)^(-0.2) \* 1 \* (O4/1)^(0.3) \* 1), 0), AR4\*1000)

**卡肉**

1. 近战武器的攻击卡肉 = 默认的攻击时长 \* 0.1
2. 远程武器的攻击卡肉 = 默认的攻击时长 \* 0.15
3. 卡肉会**累加计算**时长，例如：
1. 有一个持续3秒的攻击，中间卡肉1秒，其实会4秒打完这次攻击
2. 也因此不要配置这么长的卡肉时间
4. 卡肉不会影响STUN的动作，如果一个硬直动作持续0.5s，卡肉0.1s，硬直会在0.5s后结束
5. 卡肉会影响移动

**描述文本**

游戏中的几种技能文本的写法在用于不同位置，有明确、不同的含义

**开发配置表**

**预设以下变量：**

- 武器矫正前伤害：Dpre
- 武器校正后伤害：Dpos
- 技能校正后伤害：Dspe
- 技能没有矫正前伤害，但用该描述区分打出去的伤害和打到身上后进行了伤害减免后的伤害
- 武器击中伤害：Dwhit
- 技能击中伤害：Dshit

**技能描述规则：**

1. **[引用对象]**伤害增加**{x%}**：伤害合并，引用对象可以是：
- 武器攻击：引用武器校正后伤害Dpos
- (武器/智脑)技能：引用技能校正后伤害Dspe
- **文本样式：武器攻击**伤害增加**35%**
2. **[引用对象]**造成额外**{x%}[伤害属性]**伤害：伤害拆分，引用对象可以是：
- 武器攻击：引用武器校正后伤害Dpos
- (武器/智脑)技能：引用技能校正后伤害Dspe
- **文本样式：武器攻击**造成额外**35%电磁**伤害
- 游戏中可能存在遥远的引用文本，例如：“**武器攻击***命中时，有概率产生一道连锁闪电，*造成额外{50%}**电磁**伤害”，其含义完全等价
3. 基于**[引用对象]**实际造成的伤害，额外造成**{x%}[伤害属性]**伤害：伤害拆分，引用对象可以是：
- ：引用任意击中伤害Danyhit
- 武器攻击：引用武器击中伤害Dwhit
- (武器/智脑)技能：引用技能击中伤害：Dshit
- **文本样式：**基于**武器攻击**实际造成的伤害，额外造成**35%电磁**伤害
4. 造成**{x%}[引用对象][伤害属性]**伤害：伤害合并，引用对象可以是：
- 武器矫正前伤害：Dpre
- 属性：Attribute
- **文本样式：**造成**120💪电磁**伤害

伤害属性/异常积蓄/冲击伤害/冲击等级这四个类目都可以选择：

1. 继承自引用对象，如果使用该选择，则可以配置数值继承比例/增减
2. 独立配置，也可以不启用；合并伤害的效果不能修改伤害属性；游戏中不允许造成无属性伤害

**游戏内显示**

1. **黑色粗体**：玩家行动或者重要引用对象
2. **绿色粗体**：在游戏中是技能**注释**的数值字段
3. **黄色粗体**：在游戏中是技能**正文**的数值字段
