
**描述规则**

```在我们开始整理数值文档前，我们先定义一些文字规则 @Cade@Decay```
我草拟了如下描述规则，以便于统一语法表达、简化阅读、规范化数值设计。

该标识仅用于文档撰写，在游戏中，我们会使用其他方式（如加粗、彩色字体）标识特殊文字，但分类组别会与文档保持一致。

**效果描述**

1. 针对特殊**动词**，使用[[ ]]标识。同类动作应该保持完全一致的用词，特殊名词列表如下：
1. [[武器攻击]] / weapon attack
2. [[武器技能]] / weapon skill
3. [[智脑技能]] / cybain skill
4. [[冲刺]] / dash
5. [[奔跑]] / run
6. [[切换武器]] / switch weapon
7. [[装弹]] / reload
8. [[移动]] / move
9. [[交互]] / interact
10. 该列表（以及规则）仅包含所有1级动作，如[[交互]]应该包含“消耗信用点”、“打印武器”之类的二级动作，不适用此规则。
2. 针对**变量**，使用{ }标识。每个可升级的技能应该包含1~3个变量。
1. 变量在文本描述时应遵循尽量简洁的原则。
1. 也即如果能写成：
1. 获得额外{子弹}。
2. 就不应该写成：
1. 获得额外{几颗}子弹。
2. 变量可以对应多个技能词汇，例如：
1. [[武器攻击]]对敌人造成伤害后，对敌人及其周边造成{二次伤害}。
1. 二次伤害间隔：2s；二次伤害：150；二次伤害半径：2m
2. 上述“二次伤害：150”在数值描述部分会详细讲述书写规则，此处仅举例玩家能看到的内容
2. 当然，上述技能也可以标识为：
1. [[武器攻击]]对敌人造成伤害{后}，对敌人及其{周边}造成{二次伤害}。
3. 但这种撰写方法会使标识变得比较啰嗦，因此推荐**第一种**，以突出**“描述是针对哪个对象”**。
3. 针对**特殊词汇**，使用\*\* \*\*标识。特殊词汇包含DEBUFF、BUFF、规则，会在打印机右侧列出展示。
1. 例如：
1. [[武器攻击]]有{概率}造成\*\*暴击\*\*。
2. \*\*暴击\*\*：造成{+250%}伤害。
2. 上述例子还造成了一个特殊情况：特殊词汇中附带了变量。针对所有特殊词汇的变量，我们应该全局更改，也即：
1. 玩家同时只会造成一个比例的\*\*暴击\*\*伤害
4. 重要❗@Cade@Decay针对**伤害加成**。
1. 永远使用“+”来描述比例附加值。我们在配置时，也一律配置加值，也即：
1. 当玩家基础伤害为100时，造成“+200%”伤害会对敌人造成300伤害。
2. 当玩家基础伤害为100时，造成“+250”伤害会对敌人造成350伤害。
3. 当玩家基础伤害为100，同时拥有上述两个效果时，会对敌人造成550伤害。
4. 而在技能代码配置中也会配置200作为比例、250作为固定值数值。
5. 针对相同的行为，应该尽可能保持一致的描述和文字结构。
