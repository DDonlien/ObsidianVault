- **小球和迷宫转速的物理模拟问题 -> 解决**
1. 由于复杂的轨道特征，迷宫必须被设置为无物理模拟的moving actor
2. 基于这个前提，迷宫的旋转（角速度和角加速度）只会和controller输入设计有关，和小球在迷宫中的位置无关，小球只会基于碰撞被迷宫带动。
3. 因此，当迷宫旋转时，如果小球在迷宫最接近中央的位置，则线速度会非常小，而当小球非常接近迷宫边缘时，线速度则会非常大。此时由于小球还有物理模拟，小球的惯性会变得极其不可控：在直径比较小的迷宫中小球惯性会比较小，而在大的迷宫中，小球的惯性就会随着在迷宫中的位置变大变小，使操作变得很不顺手
4. 这并不符合现实世界的物理情况：在现实世界中如果转动这样的迷宫，小球在迷宫中的位置带来的力臂变化会带来阻力矩变化，但由于（1）所描述的前提，我们无法使用模拟物理的迷宫+torque旋转来解决这个问题（这么做计算开销也会过大）

**一种惯性问题的解决方法**

1. 我现在想到使用actor之间的碰撞来解决这个问题
2. 小球和迷宫轨道会产生至多4个碰撞点（3个点锁定碰撞面，4个点锁定移动，即使是管道中应该也不会全接触）大部分时候会是0～4个，我们应该能获得这个点的世界坐标
3. 通过小球的球心世界坐标和每个点的坐标，能获得一个矢量，也就是最多4个矢量
4. 当迷宫旋转时，事实上会实时存在一个以旋转中点为中点的旋转坐标系，迷宫的行为可以被视作形如围绕一个本身也在旋转的坐标系的某一轴旋转
5. 实时计算小球碰撞的每个矢量在这个旋转坐标系3轴的投影
6. 每个矢量在每个方向上的投影的长度和，与小球会施加给迷宫的阻力矩分量成正比
7. 我们会用这个投影和，在3个旋转方向上让迷宫旋转适度减速
8. 最终目标是，当控制器“尝试控制迷宫定速旋转时”，恰当的减速迷宫，让小球总是有同样的线速度
